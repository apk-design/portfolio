Designing an On-Site Job Application & Safety Assessment Platform

This project was created for a venture capital firm's portfolio companies with high-volume frontline roles. The goal was to build an on-site job application system designed for walk-in candidates. The platform streamlined the application process and embedded a mandatory safety-oriented quiz to help identify candidates with strong situational awareness and responsibility.

Store UX Redesign

I redesigned the in-game Store to support clearer navigation, stronger merchandising, and a more efficient purchasing flow. The goal was to give players a clean, modern interface with intuitive browsing, while simultaneously preparing the UI for full localization and accessibility compliance across multiple languages.

Project Goals

  • Strengthen Store clarity, hierarchy, and visual merchandising.
  • Support multiple monetization categories (bundles, featured, rotating items).
  • Prepare the UI for localization in up to 12+ languages.
  • Establish a predictable and scalable card system for item previews.
  • Ensure accessibility compliance (contrast, scale, label hierarchy).

Problems Identified

  • Category structure did not scale with future content.
  • Item cards varied visually, creating inconsistent browsing.
  • Text did not reflow properly in long-language translations.
  • Limited room for accessibility-safe components (AT labels, contrast ratios, etc).
  • Purchasing flow required too many steps when previewing items.

Localization & Accessibility Considerations

I produced a detailed localization callout list to help ensure the new Store design would remain stable across all supported languages. This included:

Localization Requirements

  • Item names must support 30‚Äì45% text expansion.
  • Price tags must support right-to-left (RTL) formatting.
  • Button labels must allow dynamic resizing + multiline behavior.
  • Promo banners must maintain aspect ratios regardless of text length.

Accessibility Requirements

  • Minimum 4.5:1 contrast ratio for all text-based UI.
  • Minimum 44px touch targets for all action buttons.
  • ARIA labels defined for item cards, currency indicators, and purchase buttons.
  • Reduced-motion compatibility for animated banners.

Final Store UX Structure

The final structure allocates clear hierarchical divisions while remaining flexible for future content updates. This design ensured that new bundles, skins, or promos could be added without needing a redesign.

  • Top Merchandising Row: Featured bundles or limited-time offers.
  • Modular Item Grid: Supports dynamically switching between 2‚Äì5 columns.
  • Robust Preview Pane: Shows item skins, animations, variants, and flavor text.
  • Streamlined Purchase Flow: 2-click checkout with pre-confirmation checks.

Implementation Notes

After wireframes were approved, I worked directly with UI artists and engineering to finalize:

  • Safe zone rules for text and images
  • Iconography guidelines
  • Button behavior (hover, focus, disabled states)
  • Localization tokens for dynamic data
