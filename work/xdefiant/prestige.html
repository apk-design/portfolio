Objective

Our goal was to create a repeatable, rewarding meta-progression path that:

Foundations & UX Principles

Where I Started

The original scope for Prestige was modest. As the system matured, I collaborated with our System Designer on competitive analysis across prestige systems in Call of Duty and Overwatch.

We compared:

  • Entry points
  • XP pacing
  • Reward types
  • User motivation patterns
  • Navigation + space constraints in UI

UX Principles I Pushed For

  • Hick‚Äôs Law ‚Äî reduce over-choice and keep navigation focused
  • Familiarity Bias ‚Äî draw from successful prestige patterns players understand

  • Investment ‚Äî re-earning items increases emotional buy-in
  • Temptation Bundling ‚Äî pair grind with highly desirable unlocks

The System at a Glance

First Shot at the Flow

I created the end-to-end Prestige journey and mapped where it lived in the broader UI ecosystem.

  1. Complete Core Challenges
  2. Acquire XP
  3. Acquire All Core Gear
  4. Complete the Prestige Track
  5. Initiate Prestige (reset + reward)

How the System Worked

Each Prestige tier unlocked gear or a token. Tokens activated unlockable weapons from the ‚Äúvault." The final unlock delivered remaining owned gear so the player‚Äôs prior efforts still mattered.

  • Familiarity ‚Äî linear track, known beats
  • Investment ‚Äî offering a reset as a choice deepened commitment
  • Fresh Start ‚Äî returning to starter weapons reframed the grind

User Testing

Study Design

I partnered with our Game Designer and a Tech Artist to run ten qualitative sessions with participants who understood the game but not the Prestige system.

Key Findings

  • Prestige placement made sense in the progression area
  • Testers didn‚Äôt understand Prestige immediately, but found the Info screens quickly
  • Some wanted more explicit info on the warning screen
  • Unlock CTAs worked well ‚Äî intuitive once found
  • Testers skimmed onboarding after seeing it once (clear ‚Üí concise copy was critical)
  • ‚ÄúAcknowledge‚Äù CTA was legible

Why Testing Was King

Testing provided the evidence needed to refine copy, simplify warning flows, and defend choices under tight constraints:

  • Live release deadlines forced UX trade-offs
  • Scope cuts shaped what could ship at launch
